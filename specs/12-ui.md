# Spec 12 — Local UI (Dashboard)

**Status:** Draft
**Cites:** OBJ-1 (local only), CON-3 (WSL2) (spec `00`); KPI-05, NFR UI-responsiveness; GAPS G-18 (explainability).
**Depends on:** `01-architecture` (Axum in-process, `BrainStatus` watch), `02-memory` (browse), `04` (queue view), `05/06` (council/route visibility).

---

## 1. Shape

- **U1** — Axum server inside Brain (spec 01 R4), bound to **`127.0.0.1` only** (OBJ-1 — never externally reachable). One port (default 4321). Static SPA + WebSocket; no build-time cloud dependency.
- **U2 — Live state via `watch::BrainStatus`** (spec 01 R8): current task, RAM vs 22 GB ceiling (KPI-05), queue depth, active route, council breaker states, degraded-mode banners (spec 09 H12). WebSocket push; UI render latency target ≤100 ms (draft NFR-M5).

## 2. Panels

| Panel | Shows | Source |
|---|---|---|
| Chat | conversation, streaming tokens, multimodal input (text/image/audio) | inference (spec 03) |
| Activity feed | live ledger events, filterable by kind/actor | `events` (spec 02) |
| Memory browser | OKF docs (status-colored: verified/draft/disputed/superseded), links graph, search | `okf_documents`, RAG |
| Queue | jobs (queued/running/failed/quarantined), worker states | `jobs` (spec 04) |
| Router/Council | why each route chosen, council votes + dissent, cost meter vs ceiling | `decisions`, `rewards` |
| Metrics | tok/s, RAM, local-first ratio, self-heal rate, regret curve, **learning-velocity trends (KPI-10: eval-score slope, time-to-heal trend, escalation-rate decay, observation ACTIONED rate — the growth chart, not just the height)** | eval/metrics log (spec 14), `procedural_obs` (spec 10) |
| Agents | agent_runs, briefs/handoffs, diffs, merge-gate status | `agent_runs` (spec 08) |

## 3. Explainability — "why did you say that?" (GAPS G-18)

- **U3** — Every user-facing answer carries a `trace_id`. An **Explain** action pulls the full causal event subtree (via `parent_id`, spec 02): which chunks retrieved + their scores, route chosen + why (features + prior + exploration flag), council verdicts if any, sources cited, cost, provenance mix. Nearly free — the ledger already stores all of it.
- **U4** — This is the debugging + audit surface: a confidently-wrong answer becomes fully reconstructable, and KPI-06 fact audits (spec 05) start here.

## 4. Controls (gated)

- **U5** — Approve/deny confirmations (spec 11 S3): irreversible actions, self-mod activations, over-threshold spend surface here for the human. Durable authorization (config) suppresses repeat prompts for an exact action class.
- **U6** — Read-only by default for state; mutations (pin a doc, quarantine a source, retry a job, kill a run, force degraded mode) are explicit, logged, and never bypass the harness (spec 07).

## 5. Acceptance Criteria / Test Anchors

- [ ] T1: server binds loopback only; a non-loopback request is refused. (U1, OBJ-1)
- [ ] T2: RAM approaching 22 GB → live banner + panel reflect the pressure state within one sample tick. (U2, KPI-05)
- [ ] T3: Explain on an answer reconstructs its full trace: retrieved chunks + scores, route + reason, sources, cost. (U3, G-18)
- [ ] T4: degraded mode (e.g., model down → council-only) shows a named banner, not a silent change. (U2, spec 09 H12)
- [ ] T5: an irreversible action waits for UI confirmation unless durably authorized. (U5, spec 11 S3)
- [ ] T6: a UI-initiated mutation (retry job) goes through the harness dispatcher and lands a ledger event. (U6)
